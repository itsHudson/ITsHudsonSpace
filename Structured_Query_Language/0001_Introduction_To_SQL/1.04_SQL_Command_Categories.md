# 1.4 SQL Command Categories

---

## 1.4.1 Introduction to SQL Command Categories

---

### 1.4.1.1 What are SQL Command Categories

SQL contains many commands.

To make SQL easier to organize and learn:

- SQL commands are grouped into categories

Each category has a different responsibility.

Examples:

| Category | Purpose |
|---|---|
| DDL | Define database structures |
| DML | Manipulate table data |
| DQL | Query and retrieve data |
| DCL | Control user permissions |
| TCL | Control transactions |

---

### 1.4.1.2 Why SQL Commands are Categorized

Command categories help developers:

- Organize SQL operations
- Understand database workflows
- Separate responsibilities
- Improve database management

---

### 1.4.1.3 Real-World Analogy

Think of a database system like:

- A company office building

Different SQL command categories behave like different departments.

| Department | SQL Category | Responsibility |
|---|---|---|
| Construction department | DDL | Build and modify structures |
| Office workers | DML | Work with records |
| Information desk | DQL | Retrieve information |
| Security department | DCL | Manage permissions |
| Recovery department | TCL | Handle transaction recovery |

---

### 1.4.1.4 Overview of SQL Categories

| Category | Full Name | Main Function |
|---|---|---|
| DDL | Data Definition Language | Create and manage structures |
| DML | Data Manipulation Language | Add and modify data |
| DQL | Data Query Language | Retrieve data |
| DCL | Data Control Language | Control permissions |
| TCL | Transaction Control Language | Manage transactions |

---

## 1.4.2 DDL (Data Definition Language)

---

### 1.4.2.1 What is DDL

DDL stands for:

- Data Definition Language

DDL commands are used to:

- Create structures
- Modify structures
- Remove structures

DDL mainly works with:

| Database Object | Example |
|---|---|
| Tables | students |
| Databases | school_db |
| Views | student_view |
| Indexes | idx_student |

---

### 1.4.2.2 Common DDL Commands

| Command | Purpose |
|---|---|
| CREATE | Create new objects |
| ALTER | Modify objects |
| DROP | Remove objects |
| TRUNCATE | Remove all rows |
| RENAME | Rename objects |

---

## 1.4.2.3 CREATE Command

---

### 1.4.2.3.1 What is CREATE

The CREATE command is used to:

- Create new database objects

Examples:

- Create database
- Create table
- Create view

---

### 1.4.2.3.2 CREATE TABLE Syntax

#### General Syntax

```sql
CREATE TABLE table_name (
    column1 datatype constraint,
    column2 datatype constraint,
    column3 datatype constraint
);
```

---

### 1.4.2.3.3 CREATE TABLE Example

#### Example

```sql
CREATE TABLE students (
    student_id INT,
    student_name VARCHAR(100),
    age INT
);
```

---

### 1.4.2.3.4 Step-by-Step Explanation

| Component | Explanation |
|---|---|
| CREATE TABLE | Starts table creation |
| students | Table name |
| student_id INT | Integer column |
| student_name VARCHAR(100) | Text column |
| age INT | Integer column |

---

### 1.4.2.3.5 Result After CREATE

#### students Table Structure

| student_id | student_name | age |
|---|---|---|
| Empty | Empty | Empty |

The structure exists even without data.

---

## 1.4.2.4 ALTER Command

---

### 1.4.2.4.1 What is ALTER

The ALTER command is used to:

- Modify existing structures

Examples:

| Modification | Example |
|---|---|
| Add column | ADD COLUMN |
| Remove column | DROP COLUMN |
| Change datatype | MODIFY |
| Rename column | RENAME COLUMN |

---

### 1.4.2.4.2 ALTER TABLE Syntax

#### General Syntax

```sql
ALTER TABLE table_name
ADD column_name datatype;
```

---

### 1.4.2.4.3 ALTER TABLE Example

#### Example

```sql
ALTER TABLE students
ADD email VARCHAR(100);
```

---

### 1.4.2.4.4 Before and After Example

#### Before ALTER

| student_id | student_name | age |
|---|---|---|
| 1 | Alice | 20 |

#### After ALTER

| student_id | student_name | age | email |
|---|---|---|---|
| 1 | Alice | 20 | NULL |

---

## 1.4.2.5 DROP Command

---

### 1.4.2.5.1 What is DROP

The DROP command permanently removes:

- Tables
- Databases
- Views
- Indexes

---

### 1.4.2.5.2 DROP TABLE Syntax

#### General Syntax

```sql
DROP TABLE table_name;
```

---

### 1.4.2.5.3 DROP TABLE Example

#### Example

```sql
DROP TABLE students;
```

---

### 1.4.2.5.4 Important Warning

DROP removes:

- Table structure
- Table data
- Relationships
- Constraints

Everything is permanently deleted.

---

### 1.4.2.5.5 Beginner Mistakes

| Mistake | Explanation |
|---|---|
| Confusing DROP and DELETE | DROP removes structure |
| Running DROP accidentally | Can destroy production systems |
| No backup before DROP | Dangerous in real systems |

---

## 1.4.2.6 TRUNCATE Command

---

### 1.4.2.6.1 What is TRUNCATE

The TRUNCATE command removes:

- All rows from a table

However:

- The table structure remains

---

### 1.4.2.6.2 TRUNCATE Syntax

#### General Syntax

```sql
TRUNCATE TABLE table_name;
```

---

### 1.4.2.6.3 TRUNCATE Example

#### Example

```sql
TRUNCATE TABLE students;
```

---

### 1.4.2.6.4 DELETE vs TRUNCATE

| Feature | DELETE | TRUNCATE |
|---|---|---|
| Removes rows | Yes | Yes |
| Removes structure | No | No |
| WHERE support | Yes | Usually No |
| Speed | Slower | Faster |

---

### 1.4.2.6.5 Real-World Analogy

| Action | Meaning |
|---|---|
| DELETE | Remove selected files |
| TRUNCATE | Empty entire cabinet instantly |

---

## 1.4.3.4 UPDATE Command

---

### 1.4.3.4.1 What is UPDATE

The UPDATE command modifies:

- Existing rows

---

### 1.4.3.4.2 UPDATE Syntax

#### General Syntax

```sql
UPDATE table_name
SET column_name = value
WHERE condition;
```

---

### 1.4.3.4.3 UPDATE Example

#### Example

```sql
UPDATE students
SET age = 21
WHERE student_id = 1;
```

---

### 1.4.3.4.4 Before and After Example

#### Before UPDATE

| student_id | student_name | age |
|---|---|---|
| 1 | Alice | 20 |

#### After UPDATE

| student_id | student_name | age |
|---|---|---|
| 1 | Alice | 21 |

---

### 1.4.3.4.5 Critical Beginner Mistake

#### Dangerous Query

```sql
UPDATE students
SET age = 21;
```

Without WHERE:

- Every row gets updated

---

### 1.4.3.5.1 What is DELETE

The DELETE command removes:

- Rows from a table

---

### 1.4.3.5.2 DELETE Syntax

#### General Syntax

```sql
DELETE FROM table_name
WHERE condition;
```

---

### 1.4.3.5.3 DELETE Example

#### Example

```sql

```

---

### 1.4.3.5.4 Important Difference

| Command | Removes |
|---|---|
| DELETE | Rows only |

### 1.4.3.5.5 Dangerous Beginner Mistake

#### Dangerous Query


## 1.4.3.6 MERGE Command

---

### 1.4.3.6.1 What is MERGE

The MERGE command combines:

- INSERT
- UPDATE
- DELETE

into one operation.

---

### 1.4.3.6.2 Why MERGE is Useful

MERGE is commonly used for:

- Data synchronization
- ETL pipelines
- Data warehousing

---

### 1.4.3.6.3 MERGE Syntax

#### General Syntax

```sql
MERGE INTO target_table
USING source_table
ON condition
WHEN MATCHED THEN
    UPDATE SET column_name = value
WHEN NOT MATCHED THEN
    INSERT (column1, column2)
    VALUES (value1, value2);
```

---

### 1.4.3.6.4 MERGE Example

#### Example

```sql
MERGE INTO students_target
USING students_source
ON students_target.student_id = students_source.student_id
WHEN MATCHED THEN
    UPDATE SET students_target.student_name = students_source.student_name
WHEN NOT MATCHED THEN
    INSERT (
        student_id,
        student_name
    )
    VALUES (
        students_source.student_id,
        students_source.student_name
    );
```

## 1.4.6 TCL (Transaction Control Language)

---

### 1.4.6.1 What is TCL

TCL stands for:

- Transaction Control Language

TCL manages:

- Database transactions

---

### 1.4.6.2 What is a Transaction

A transaction is:

- A group of SQL operations treated as one unit

Example:

| Step | Action |
|---|---|
| Step 1 | Deduct money |
| Step 2 | Add money |
| Step 3 | Confirm transaction |

If one step fails:

- Entire transaction should fail

---

### 1.4.6.3 Common TCL Commands

| Command | Purpose |
|---|---|
| COMMIT | Save transaction |
| ROLLBACK | Cancel transaction |
| SAVEPOINT | Create checkpoint |

---

## 1.4.6.6 SAVEPOINT Command

---

### 1.4.6.6.1 What is SAVEPOINT

A SAVEPOINT creates:

- A checkpoint inside a transaction

---

### 1.4.6.6.2 Why SAVEPOINT is Useful

SAVEPOINT allows:

- Partial rollback

instead of:

- Rolling back entire transaction

---

### 1.4.6.6.3 SAVEPOINT Syntax

#### General Syntax

```sql

```

---

### 1.4.6.6.4 SAVEPOINT Example

#### Example

```sql

```

---

### 1.4.6.6.5 Rollback to SAVEPOINT Syntax

#### General Syntax

```sql

```

---

### 1.4.6.6.6 Rollback to SAVEPOINT Example

#### Example

```sql

```

## 1.4.7 Complete SQL Command Workflow

---

### 1.4.7.1 Real-World SQL Flow

#### Step 1 — Create Table (DDL)

```sql
CREATE TABLE employees (
    employee_id INT,
    employee_name VARCHAR(100)
);
```

#### Step 2 — Insert Data (DML)

```sql
INSERT INTO employees (
    employee_id,
    employee_name
)
VALUES (
    1,
    'Alice'
);
```

#### Step 3 — Retrieve Data (DQL)

```sql
SELECT *
FROM employees;
```

#### Step 4 — Grant Access (DCL)

```sql
GRANT SELECT
ON employees
TO analyst_user;
```

#### Step 5 — Save Transaction (TCL)

```sql
COMMIT;
```

## 1.4.8 Beginner Mistakes Across SQL Categories

---

### 1.4.8.1 Common Beginner Mistakes

| Mistake | Explanation |
|---|---|
| Forgetting WHERE in UPDATE | Updates all rows |
| Forgetting WHERE in DELETE | Deletes all rows |
| Confusing DROP and DELETE | DROP removes structure |
| Running COMMIT too early | Makes mistakes permanent |
| Ignoring permissions | Security risk |
| Using TRUNCATE carelessly | Removes all rows instantly |

# 1.1 SQL Fundamentals

Before learning joins, subqueries, indexing, optimization, or database engineering, you must first understand what SQL actually is and where it fits in real systems.

Many beginners fail because they jump straight into writing queries without understanding:

- what a database is
- why SQL exists
- where SQL is used
- how SQL works behind the scenes
- why companies heavily rely on SQL

This section builds that foundation.

---

# 1.1.1 What is SQL?

---

## Definition

SQL stands for:

**Structured Query Language**

It is a language used to communicate with relational databases.

A database stores information in an organized way.

Think of SQL as a language you use to "talk" to the database.

You can tell the database:

- store data
- retrieve data
- update data
- delete data
- create tables
- control access

---

## Real Life Analogy

Imagine a school office storing student records manually.

They have cabinets:

- Student records
- Teacher records
- Exam records

Inside each cabinet:

Folders

Inside each folder:

Student details

Database works similarly:

| Real Life | Database |
|------------|------------|
| Cabinet | Database |
| Folder | Table |
| Paper record | Row |
| Information field | Column |

---

## Example Table

Student table:

| student_id | name | age | course |
|------------|--------|------|---------|
| 1 | John | 20 | IT |
| 2 | Sarah | 21 | Business |

---

## What SQL Can Do

| Task | SQL Command Example |
|--------|--------------------|
| Retrieve data | `SELECT` |
| Insert new data | `INSERT` |
| Update data | `UPDATE` |
| Delete data | `DELETE` |
| Create tables | `CREATE TABLE` |
| Remove tables | `DROP TABLE` |

---

## Step-by-Step: How SQL Works

Step 1 → User writes SQL query

```sql
SELECT name
FROM students;
```

Step 2 → Database receives query

Step 3 → SQL parser checks syntax

Step 4 → Database searches table

Step 5 → Matching records returned

---

## Output Example

| name |
|--------|
| John |
| Sarah |

---

## Real World Example

When Shopee shows your order history:

- SQL retrieves your orders
- SQL retrieves payment records
- SQL retrieves shipping records

---

## Beginner Mistakes

SQL is only the language.

Examples of databases:

---

## Key Takeaway

SQL helps humans communicate with databases.

# 1.1.4 SQL vs Programming Languages

Many beginners confuse SQL with Python, Java, etc.

They are different.

---

## SQL = Declarative Language

You tell database:

**WHAT you want**

Example:

```sql
SELECT name
FROM employees;
```

You do not explain how database should search.

Database decides.

---

## Programming Languages = Procedural

Examples:

- Python
- Java
- JavaScript
- C++

You tell computer:

**HOW to do something**

Example:

```python
for employee in employees:
    print(employee.name)
```

---

## Comparison Table

| Feature | SQL | Programming Languages |
|----------|------|------------------------|
| Purpose | Manage data | Build applications |
| Type | Declarative | Procedural/OOP |
| Example | SQL | Python |

---

## Real Workflow Example

Login system:

Frontend → login page

Backend → verifies login

SQL → checks user database

---

## Key Takeaway

SQL handles data.

Programming languages build applications.

# 1.1 SQL Fundamentals

## 1.1.1 What is SQL

---

### 1.1.1.1 Definition of SQL

SQL stands for:

- Structured Query Language

SQL is a specialized language used to:

- Store data
- Retrieve data
- Manage data
- Modify data
- Control database access
- Perform data analysis

SQL is mainly used with:

- Relational Database Management Systems (RDBMS)

Examples:

| Database System | Type |
|---|---|
| MySQL | Open-source relational database |
| PostgreSQL | Advanced open-source relational database |
| Microsoft SQL Server | Enterprise relational database |
| Oracle Database | Enterprise database |
| SQLite | Lightweight embedded database |

---

### 1.1.1.2 Simple Real-World Analogy

Think of a database like:

- A digital filing cabinet

Think of SQL like:

- The language used to communicate with the filing cabinet

Example:

| Action | Real Life | SQL Equivalent |
|---|---|---|
| Find student record | Search a file folder | SELECT |
| Add new employee | Add new document | INSERT |
| Update customer phone number | Edit document | UPDATE |
| Remove old order | Throw away document | DELETE |

---

### 1.1.1.3 What SQL Can Do

SQL can perform many operations.

Common SQL Operations

| Operation | Purpose |
|---|---|
| CREATE | Create databases and tables |
| INSERT | Add data |
| SELECT | Retrieve data |
| UPDATE | Modify data |
| DELETE | Remove data |
| ALTER | Change table structure |
| DROP | Delete database objects |
| GRANT | Control permissions |

---

### 1.1.1.4 Example of SQL Syntax

#### Create a Table

```sql
CREATE TABLE students (
    student_id INT,
    student_name VARCHAR(100),
    age INT
);
```

#### Insert Data

```sql
INSERT INTO students (
    student_id,
    student_name,
    age
)
VALUES (
    1,
    'Alice',
    20
);
```

#### Retrieve Data

```sql
SELECT *
FROM students;
```

---

### 1.1.1.5 How SQL Works Step-by-Step

When you run an SQL query:

```sql
SELECT * FROM students;
```

The database system performs these steps:

| Step | Description |
|---|---|
| Step 1 | SQL query is sent to database |
| Step 2 | Database parses the query |
| Step 3 | Database validates table and column names |
| Step 4 | Database optimizer creates execution plan |
| Step 5 | Database engine retrieves data |
| Step 6 | Results are returned to user |

---

### 1.1.1.6 Important Characteristics of SQL

| Characteristic | Explanation |
|---|---|
| Declarative Language | You describe WHAT you want, not HOW to get it |
| Standardized | ANSI and ISO standards exist |
| Table-Based | Works mainly with relational tables |
| Powerful Querying | Supports filtering, sorting, joining, aggregation |
| Widely Used | Used in almost every industry |

---

### 1.1.1.7 Beginner Mistakes

| Mistake | Explanation |
|---|---|
| Thinking SQL is a programming language | SQL is mainly a query language |
| Forgetting semicolons | Some systems require query termination |
| Confusing tables and databases | Tables exist inside databases |
| Ignoring NULL values | NULL behaves differently from normal values |
| Writing queries without WHERE | Can accidentally update/delete all rows |

## 1.1.4 SQL vs Programming Languages

---

### 1.1.4.1 Core Difference

SQL is mainly:

- A declarative query language

Programming languages are mainly:

- Procedural or object-oriented languages

---

### 1.1.4.2 Comparison Table

| Feature | SQL | Programming Languages |
|---|---|---|
| Main Purpose | Query databases | Build applications |
| Execution Style | Declarative | Procedural/Object-oriented |
| Focus | Data operations | Full system logic |
| Examples | SQL | Python, Java, C++ |
| Loops | Limited | Full support |
| UI Development | No | Yes |
| File Handling | Limited | Full support |

---

### 1.1.4.3 Real-World Workflow

A backend application usually combines both.

Example Flow

| Step | Technology |
|---|---|
| User clicks login | Frontend |
| Backend receives request | Java/Python/Node.js |
| Backend queries database | SQL |
| Database returns data | SQL Database |
| Backend sends response | Backend Language |

---

### 1.1.4.4 Example Integration

#### Python + SQL

```python
query = "SELECT * FROM users"
```

The programming language controls:

- Logic
- API
- Authentication
- User interaction

SQL controls:

- Data retrieval

## 1.1.5 SQL vs NoSQL

---

### 1.1.5.1 What is NoSQL

NoSQL means:

- Not Only SQL

NoSQL databases are designed for:

- Flexible data structures
- Massive scalability
- Distributed systems

---

### 1.1.5.2 SQL Database Structure

SQL databases use:

- Tables
- Rows
- Columns
- Fixed schema

Example:

| id | name | age |
|---|---|---|
| 1 | Alice | 20 |

---

### 1.1.5.3 NoSQL Database Structure

NoSQL databases may use:

- Documents
- Key-value pairs
- Graphs
- Wide-column storage

Example JSON document:

```json
{
  "id": 1,
  "name": "Alice",
  "skills": ["Python", "SQL"]
}
```

---

### 1.1.5.4 Comparison Table

| Feature | SQL | NoSQL |
|---|---|---|
| Structure | Structured tables | Flexible structure |
| Schema | Fixed schema | Dynamic schema |
| Relationships | Strong relational support | Limited relational support |
| Scalability | Vertical scaling common | Horizontal scaling common |
| Transactions | Strong ACID support | Often eventual consistency |
| Best For | Structured business data | Large distributed systems |

---

### 1.1.5.5 Examples of NoSQL Databases

| Database | Type |
|---|---|
| MongoDB | Document |
| Redis | Key-value |
| Cassandra | Wide-column |
| Neo4j | Graph |

---

### 1.1.5.6 Beginner Mistakes

| Mistake | Explanation |
|---|---|
| Thinking NoSQL replaces SQL | Both solve different problems |
| Assuming SQL cannot scale | Modern SQL systems scale very well |
| Ignoring data relationships | SQL excels in relational data |

### 1.1.10.3 Feature Engineering with SQL

Example:

```sql

    customer_id,
    SUM(order_amount) AS total_spent,
    COUNT(order_id) AS total_orders

GROUP BY customer_id;
```

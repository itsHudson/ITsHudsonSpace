# 1.1.1 What is SQL?

## Definition

SQL stands for:

**Structured Query Language**

It is a language used to communicate with relational databases.

SQL allows users to:

- store data
- retrieve data
- update data
- delete data
- create tables
- manage permissions
- control transactions

---

## Real Life Analogy

Think of a database like a school filing system.

### Real life structure

- A **cabinet** stores all school records
- A **folder** stores one category of records
- A **paper sheet** stores one student's record
- Each field contains specific information

Example student information:

- Student ID
- Student Name
- Age
- Course

---

## Database equivalent

| Real Life Item | Database Equivalent |
|-----------------|----------------------|
| Cabinet | Database |
| Folder | Table |
| Paper Sheet | Row |
| Information Field | Column |

---

## Example Table

Student table:

| student_id | name | age | course |
|------------|------|------|---------|
| 1 | John | 20 | IT |
| 2 | Sarah | 21 | Business |

### In this table:

**Columns**

- student_id
- name
- age
- course

**Rows**

- John's record
- Sarah's record

---

## How SQL Works (Step-by-Step)

Step 1 → User writes SQL query

```sql
SELECT name
FROM students;
```

Step 2 → Database receives query

Step 3 → SQL checks query syntax

Step 4 → Database finds matching data

Step 5 → Results returned to user

---

## Output Example

| name |
|------|
| John |
| Sarah |

# 1.1.4 SQL vs Programming Languages

Many beginners confuse SQL with Python, Java, or JavaScript.

They are different.

---

## SQL

SQL tells database:

**WHAT data you want**

Example:

```sql
SELECT name
FROM employees;
```

Database decides how to retrieve it.

---

## Programming Languages

Examples:

- Python
- Java
- JavaScript
- C++

These languages tell computer:

**HOW to perform tasks**

Example:

```python
for employee in employees:
    print(employee.name)
```

---

## Comparison Table

| Feature | SQL | Programming Languages |
|----------|------|------------------------|
| Type | Declarative | Procedural/OOP |
| Focus | Data | Application logic |
| Example | SQL | Python |

---

## Real Workflow Example

Login system:

- Frontend collects login info
- Backend verifies credentials
- SQL checks database
- User logs in

---

## Key Takeaway

SQL manages data.

Programming languages build applications.

## 1.1.1 What is SQL?

SQL is the language used to communicate with relational databases.

SQL stands for:

**Structured Query Language**

SQL allows users to:

- retrieve data
- insert data
- update data
- delete data
- create database structures
- manage permissions
- control transactions

---

### 1.1.1.1 Definition

A database stores information in an organized format.

SQL acts like a communication language between humans/applications and the database.

Using SQL, we can ask the database questions like:

- "Show all students"
- "Add a new customer"
- "Update account balance"
- "Delete old records"

---

### 1.1.1.2 Real Life Analogy

Think of a database like a school office filing system.

---

#### 1.1.1.2.1 Real Life Structure

#### 1.1.1.2.2 Database Equivalent

### 1.1.1.4 What SQL Can Do

| Operation | Purpose | Example |
|------------|----------|----------|
| Retrieve Data | Read information | `SELECT` |
| Insert Data | Add records | `INSERT` |
| Update Data | Modify records | `UPDATE` |
| Delete Data | Remove records | `DELETE` |
| Create Structures | Create tables/databases | `CREATE` |
| Manage Security | Control permissions | `GRANT` |
| Manage Transactions | Save/revert changes | `COMMIT` |

---

### 1.1.1.5 How SQL Works (Step-by-Step)

Step 1 → User writes SQL query

```sql

```

Step 2 → Database receives query

Step 3 → SQL parser checks syntax

Step 4 → Database searches matching data

Step 5 → Database returns result

## 1.1.4 SQL vs Programming Languages

Many beginners confuse SQL with programming languages.

They are different.

---

### 1.1.4.1 SQL

SQL is a declarative language.

You tell database:

**WHAT you want**

Example:

```sql
SELECT name
FROM employees;
```

Database decides how to retrieve data.

---

### 1.1.4.2 Programming Languages

Examples:

- Python
- Java
- JavaScript
- C++

Programming languages tell computer:

**HOW to perform tasks**

Example:

```python
for employee in employees:
    print(employee.name)
```

---

### 1.1.4.3 Comparison Table

| Feature | SQL | Programming Languages |
|----------|------|------------------------|
| Type | Declarative | Procedural/OOP |
| Focus | Data | Application logic |
| Example | SQL | Python |

---

### 1.1.4.4 Real Workflow Example

Login workflow:

Step 1 → User enters credentials

Step 2 → Backend receives request

Step 3 → SQL checks database

Step 4 → Backend verifies password

Step 5 → User logs in

---

### 1.1.4.5 Key Takeaway

SQL manages data.

Programming languages build applications.

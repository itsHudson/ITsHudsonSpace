# 1\. What coding practice really means / 什么才叫真正的代码练习

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **Coding Practice**<br><br>**代码练习** | Coding practice means actively using programming concepts to solve tasks, not just reading or copying code.<br><br>代码练习是主动使用编程概念去解决任务，而不只是看代码或抄代码。 | You learn a concept, apply it to a small task, see what breaks, fix it, and repeat until the idea becomes usable without heavy support.<br><br>你先学一个概念，再把它用到小任务里，看看哪里坏掉、修正、重复，直到这个概念变成你能独立使用的能力。 | Example: After learning loops, you write programs that count, sum, filter, and repeat prompts instead of only rereading notes.<br><br>例子：学完循环后，你会写计数、求和、筛选和重复提示输入的程序，而不是只反复看笔记。 |
| **Practice builds skill**<br><br>**练习建立技能** | Skill comes from action with feedback, not from passive familiarity.<br><br>技能来自“行动 + 反馈”，不是来自被动熟悉感。 | Every attempt shows what you understand, forget, misapply, and need to strengthen next.<br><br>每次尝试都会给你信息：哪里懂、哪里忘、哪里用错、下一步要加强什么。 | Example: A failed list exercise shows you know indexing but not mutation or traversal yet.<br><br>例子：失败的列表练习可能说明你会索引，但还不会修改或遍历。 |
| **Practice is controlled struggle**<br><br>**练习是可控的挣扎** | Good practice feels slightly difficult because your brain is being stretched.<br><br>好的练习会有一点难，因为大脑在被拉出舒适区。 | Too easy = little growth; too hard = overload. Effective practice is in between.<br><br>太简单成长少；太难会过载。有效练习在中间。 | Example: A simple number-checker after learning `if` is good; building a full web app on day two is not.<br><br>例子：学完 if 写数字判断是合适；第二天做完整网站不合适。 |
| **Practice is not random**<br><br>**练习不能乱练** | Useful practice is targeted and structured, not blind random problems.<br><br>有用练习要有针对性和结构，不是盲目乱做题。 | Know which concept you train, why, and how it matches your level.<br><br>要知道在练什么、为什么练、与当前水平的关系。 | Example: “Today I practice loop counters and accumulation” is stronger than “random coding”.<br><br>例子：“今天练计数与累加”比“随便练代码”更有效。 |

# 2\. Why many beginners practice badly / 为什么很多初学者练得很差

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **Too much copying**<br><br>**抄太多** | Copying full solutions too early creates false progress.<br><br>太早抄完整答案，会制造假的进步感。 | Copying may finish tasks but blocks independent logic building.<br><br>抄答案能完成任务，但会阻碍独立逻辑建立。 | Example: You can copy a sum loop, but later fail to build one yourself.<br><br>例子：能抄求和循环，但自己写不出来。 |
| **Practicing without focus**<br><br>**没有焦点地练** | Random practice weakens learning because no pattern is reinforced.<br><br>乱练会削弱学习，因为没有模式被强化。 | Switching topics constantly prevents any concept from stabilizing.<br><br>频繁切换让任何概念都难以稳定。 | Example: In one session mixing loop, API, SQL, GUI is ineffective for beginners.<br><br>例子：一节同时练循环、API、SQL、GUI通常效果很差。 |
| **No review**<br><br>**没有复盘** | Practicing without review hides weak points.<br><br>不复盘会隐藏弱点。 | Mistakes repeat when not inspected.<br><br>不检查就会重复犯错。 | Example: Forgetting `int(input(...))` keeps happening if not noted.<br><br>例子：不记录就会一直忘记类型转换。 |
| **Overestimating understanding**<br><br>**高估理解** | “I saw it, so I can do it” is often false.<br><br>“看懂了就会做”常常不成立。 | Recognition is easier than production.<br><br>识别比产出容易。 | Example: You recognize list code but freeze when writing it alone.<br><br>例子：看懂列表代码，但自己写会卡住。 |

# 3\. The real goal of coding practice / 代码练习真正的目标

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **From knowledge to use**<br><br>**从知识到可用能力** | Turn “I saw this” into “I can use this”.<br><br>把“看过”变成“会用”。 | Reuse concepts across slightly different tasks.<br><br>在变化题中重复使用。 | Example: Use `if` for validation, filtering, and menus—not only scores.<br><br>例子：if 用在输入验证、筛选和菜单。 |
| **Build independence**<br><br>**建立独立性** | Reduce dependence on notes and examples over time.<br><br>逐步减少对笔记和例子的依赖。 | Need fewer hints as practice grows.<br><br>练多后提示需求减少。 | Example: From following a sample to writing your own function.<br><br>例子：从跟写到自己写函数。 |
| **Build transfer ability**<br><br>**建立迁移能力** | Apply the same idea in different contexts.<br><br>把同一概念迁移到不同场景。 | Seeing patterns across tasks deepens learning.<br><br>跨题型识别模式会更深。 | Example: A loop can count, scan strings, or process file rows.<br><br>例子：循环可用于计数、字符串、文件处理。 |
| **Build confidence through evidence**<br><br>**通过证据建立自信** | Confidence comes from repeated success, not motivation alone.<br><br>自信来自反复成功，不只是打气。 | Each solved task is proof of ability.<br><br>每题都是能力证据。 | Example: 10 small working programs > 10 watched tutorials.<br><br>例子：10 个小程序 > 10 个教程。 |

# 4\. The best order for practice after learning a topic / 学完一个知识点后的最佳练习顺序

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **Step 1: Re-type the example**<br><br>**第一步：重打例子** | Re-type the original example yourself.<br><br>把例子自己重打一遍。 | Connect “seeing” to “typing and syntax control”.<br><br>把“看懂”连接到“能输入与控制语法”。 | Example: Re-type string indexing, slicing, `.upper()`.<br><br>例子：重打字符串索引、切片和方法。 |
| **Step 2: Modify the example**<br><br>**第二步：改例子** | Change it so it is no longer identical.<br><br>把例子改掉。 | Tests understanding of behavior, not appearance.<br><br>测试对行为的理解，而不是外观。 | Example: Change 1–5 loop to print even numbers 2–10.<br><br>例子：改成输出 2–10 的偶数。 |
| **Step 3: Solve a very small task**<br><br>**第三步：小任务** | Use the concept in a new simple task.<br><br>在新但简单的任务中使用。 | Moves from imitation to independent use.<br><br>从模仿走向独立。 | Example: Write a program to classify positive/negative/zero.<br><br>例子：判断正负与零。 |
| **Step 4: Do 3–5 variations**<br><br>**第四步：3–5 个变化** | Repeat with variations.<br><br>带变化地重复。 | Same core idea with changed data/rules.<br><br>核心不变，数据或规则变化。 | Example: Counting, summing, max-finding, input repetition.<br><br>例子：计数、求和、最大值、重复输入。 |
| **Step 5: Build a mini task**<br><br>**第五步：小整合** | Combine with earlier concepts.<br><br>与前面知识整合。 | Connect pieces into usable structure.<br><br>把碎片连成结构。 | Example: Menu that loops until user stops.<br><br>例子：循环菜单直到退出。 |

# 5\. What a good daily practice session looks like / 一次好的日常练习长什么样

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **Warm-up**<br><br>**预热** | Review a small amount before new work.<br><br>先做少量复习再进入新内容。 | Reactivates memory, reduces cold start.<br><br>激活记忆，减少生疏。 | Example: Review yesterday’s loop and one exercise.<br><br>例子：回顾昨天循环与一道题。 |
| **Focused target**<br><br>**明确目标** | One main target per session.<br><br>每节一个主要目标。 | Focus strengthens one pattern deeply.<br><br>聚焦强化单一模式。 | Example: “Practice list traversal and accumulation.”<br><br>例子：练列表遍历与累加。 |
| **Short active work**<br><br>**短而主动** | Main time = writing code.<br><br>主体是写代码。 | Forces recall, syntax, logic, error handling.<br><br>逼你回忆、输出语法与处理错误。 | Example: 25–40 min coding, not just reading.<br><br>例子：25–40 分钟写代码。 |
| **Debug and reflect**<br><br>**调试反思** | Inspect what went wrong and why.<br><br>检查哪里错、为什么错。 | Turn mistakes into reusable lessons.<br><br>把错误变成可复用经验。 | Example: Note “forgot int conversion”.<br><br>例子：记录忘记类型转换。 |
| **Close with summary**<br><br>**总结收尾** | Summarize gains, gaps, next step.<br><br>总结进步、弱点、下一步。 | Builds continuity across sessions.<br><br>建立连续性。 | Example: “Loops OK, nested loops still weak.”<br><br>例子：简单循环OK，嵌套弱。 |

# 6\. What kinds of practice you should do / 你应该做哪些类型的练习

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **Reproduction practice**<br><br>**重现练习** | Rebuild what you saw without line-by-line copying.<br><br>不逐行抄，重建看过的内容。 | Bridges recognition → production.<br><br>连接识别到产出。 | Example: Recreate an `if-else` from memory.<br><br>例子：凭记忆重写 if-else。 |
| **Modification practice**<br><br>**修改练习** | Change an existing solution’s behavior.<br><br>修改已有解法。 | Understand which parts control behavior.<br><br>理解哪些代码影响行为。 | Example: Pass/fail → A/B/C/F classifier.<br><br>例子：及格改等级分类。 |
| **From-scratch practice**<br><br>**从零练习** | Solve without a full solution in front of you.<br><br>没有完整答案时自己做。 | Forces planning, recall, decomposition, debugging.<br><br>逼你做规划、回忆、拆解、调试。 | Example: Write a menu calculator from scratch.<br><br>例子：从零写菜单计算器。 |
| **Tracing practice**<br><br>**追踪练习** | Predict outputs or variable states.<br><br>预测输出或变量状态。 | Strengthens execution-flow understanding.<br><br>强化执行流程理解。 | Example: Track `total` each loop iteration.<br><br>例子：逐次追踪 total。 |
| **Debugging practice**<br><br>**调试练习** | Fix broken/incomplete code.<br><br>修复错误代码。 | Trains reading, error interpretation, precise fixes.<br><br>训练读码、解错与修正。 | Example: Fix string vs int comparison crash.<br><br>例子：修复字符串与整数比较错误。 |
| **Mini-project practice**<br><br>**小项目练习** | Combine multiple concepts into a small app.<br><br>整合多个概念。 | Connect separate topics into one flow.<br><br>把分散知识连成流程。 | Example: Student mark analyzer with input, loops, lists, conditions, functions.<br><br>例子：学生分数分析器。 |

# 10\. How to practice without copying too much / 怎样练习又不至于抄太多

| **Type**<br><br>**类型** | **Core Idea**<br><br>**核心概念** | **How it works**<br><br>**它如何运作** | **Usage / Example**<br><br>**用途 / 例子** |
|---|---|---|---|
| **Try first**<br><br>**先自己试** | Attempt before seeing solutions.<br><br>先尝试再看答案。 | Even partial attempts expose gaps.<br><br>不完整尝试也能暴露卡点。 | Example: Try max-number logic before searching.<br><br>例子：先自己写最大值逻辑。 |
| **Hints before full answers**<br><br>**先提示后答案** | Prefer hints over full solutions.<br><br>先看提示。 | Preserves your problem-solving process.<br><br>保留你的解题过程。 | Example: “Use a loop and track current max.”<br><br>例子：用循环记录最大值。 |
| **Compare after attempt**<br><br>**先做再对照** | Read solutions after honest attempt.<br><br>尝试后再看答案。 | Compare your thinking with working approach.<br><br>对照你的思路与正确方法。 | Example: Remember to initialize `max_value`.<br><br>例子：记住先初始化。 |
| **Rewrite from memory**<br><br>**看完再重写** | Rebuild after reading.<br><br>看完再重建。 | Turn seen knowledge into owned control.<br><br>把“看懂”变成“会用”。 | Example: Close solution and recreate it.<br><br>例子：关掉答案再写一遍。 |

# 12\. How to use mistakes correctly / 怎样正确利用错误

|---|---|---|---|
| **Mistakes are data**<br><br>**错误是数据** | Errors reveal misunderstandings.<br><br>错误揭示误解。 | Syntax vs logic vs runtime indicate different issues.<br><br>语法/逻辑/运行时对应不同问题。 | Example: `TypeError` = incompatible types.<br><br>例子：类型不匹配。 |
